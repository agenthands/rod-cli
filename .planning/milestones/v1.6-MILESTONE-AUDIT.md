## Integration Assessment
The config spine is the integration keystone and it holds end-to-end: a single
`flag → StealthFlags → ResolveStealth (precedence) → Config.Stealth → NewContext →
consumer` path, resolved once at daemon spawn and frozen, is reused by every feature
(proxy, fingerprint, hardening toggles, humanize tuning). Each stealth assertion reads
back from the live (daemon-reused) browser via `page.Eval`, never from a Go config field
— the v1.5 "wired-but-silent" lesson is structurally honored. Harness-first ordering
held: Phase 24's offline harness gated and regression-netted every later phase.
