# Plan 18-01: Lifecycle Event Emitters

## Objective
Expose godoll browser events (OnRequest, OnResponse, OnLoad, OnDOMNodeInserted) to the plugin engine.

## Requirements Covered
- PLUG-03
- PLUG-04
- PLUG-05
- PLUG-06

## Tasks

<task>
<read_first>
- internal/plugin/engine.go
</read_first>
<action>
Create `internal/plugin/lifecycle.go`.
Define an interface `LifecycleEmitter` with methods `OnRequest`, `OnResponse`, `OnLoad`, and `OnDOMNodeInserted`.
Add a method `BindLifecycle(page *rod.Page)` to `PluginEngine` to attach CDP event listeners to the active page and forward them to JS functions defined in the plugin scripts if they exist (e.g., calling a global JS function `onRequest` or emitting an event to a global `Events` object).
</action>
<acceptance_criteria>
- `internal/plugin/lifecycle.go` exists.
- `BindLifecycle` successfully sets up `page.EachEvent` listeners.
- The engine can invoke global JS functions corresponding to the lifecycle hooks.
</acceptance_criteria>
</task>

## Verification
<must_haves>
<truths>
- Engine can define an OnRequest handler that fires before network dispatch.
- Engine can define OnLoad and OnResponse handlers that receive state.
- Engine can intercept dynamic DOM mutations (OnDOMNodeInserted).
</truths>
</must_haves>

## Artifacts this phase produces
- `internal/plugin/lifecycle.go`
- `LifecycleEmitter`
- `BindLifecycle`
