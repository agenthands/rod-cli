<objective>
Ship the four per-hook recipe plugins (PEX-02) and the copyable starter template (PEX-03) under `plugins/examples/`. Each recipe is a minimal script: one lifecycle hook + one module-level results array + one JSON getter, so a user can `plugin load` it, drive the browser, then `plugin run <getter>` to see that hook fired. The starter defines empty stubs for all four hooks plus a results accessor and must load unchanged.

All scripts mirror the existing `xss_scanner.js` shape (module-level array, lowercase hook, JSON.stringify getter) and run in the goja sandbox — no console, fs, require, or import.

Purpose: PEX-02 needs a runnable recipe per hook; PEX-03 needs a clean-loading scaffold.
Output: 4 recipe scripts + 1 starter script.
</objective>
