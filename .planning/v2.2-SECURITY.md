# v2.2 Security Review — Pi Extension

## Verdict: PASSED — no gaps, no tech_debt

## Scope

TypeScript Pi extension under `extensions/pi/`. 13 browse_* tools shelling out to
rod-cli Go binary via `pi.exec()`. Zero Go changes.

## Review findings

### 1. Input validation — ADEQUATE

| Guard | Location | Assessment |
|-------|----------|------------|
| URL format (http/https) | cli.ts:138-141 | Adequate. SSRF concerns dismissed — rod-cli IS a browser. Blocking loopback would break integration tests, detection harness, and local dev. |
| Selector non-empty | cli.ts:145-147 | Adequate. Prevents empty-string selectors from reaching rod-cli. |
| Fill text non-empty | cli.ts:149-151 | Adequate. |
| Eval expression ≤10KB | cli.ts:154-159 | Adequate. Prevents accidental megabyte JS payloads. |

### 2. Argv injection (I6) — COMPLETE

All 13 tools verified with `--` separator before user-controlled positional args.
`--submit` flag in fill_form.ts:37 correctly placed BEFORE `--`.

### 3. Binary resolution — NO PATH TRAVERSAL RISK

- `findRodCli()` uses `join(dir, BINARY_NAME)` — prevents directory traversal
- `statSync` with `throwIfNoEntry: false` — no crash on missing binary
- Returns `null` (not throw) — caller handles gracefully

### 4. Daemon lifecycle — ADEQUATE

- `session_shutdown` gated on `reason === "quit"` — no teardown on reload/fork
- Best-effort catch — daemon's own PPID polling + 15min idle timeout is safety net
- `session_start` only runs `--version` (read-only) — no browser launched

### 5. pi.exec usage — CORRECT

- AbortSignal propagated to every call
- Per-command timeouts (goto 60s, snapshot 15s, close 5s, etc.)
- `--raw` prepended for machine-readable output

### 6. Trust boundaries — ACCEPTABLE

- Extension runs in-process with Pi (all Pi extensions do — this is Pi's model)
- LLM can drive a browser through typed tools — the intended function
- No file system access, no network listeners, no secrets management
- No elevation path beyond what the user explicitly requested

## Findings: NONE
