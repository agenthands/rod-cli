<objective>
Replace godoll's no-op canvas/audio noise with a **seeded, stable-per-session** implementation, and add the `SpoofCanvas` gate so rod-cli can toggle it. The `ApplyCanvasNoise` stub (and the equally-empty `scriptCanvasNoise` const + the `Math.random()`-based `scriptAudioContextNoise`) become deterministic PRNG generators keyed off an `EvasionManager.noiseSeed`. Same seed → identical canvas/audio hashes across re-reads (the HARDEN-02 stability invariant); a different session seed → different hashes.

Purpose: This is the godoll engine half of HARDEN-02. rod-cli Plan 03 sets the seed (`em.SetNoiseSeed`) and gate (`profile.SpoofCanvas`) from `cfg.Stealth.CanvasNoise`; the harness (Plan 04) asserts hash stability. godoll is consumed via `replace => ../godoll` and its changes stay UNCOMMITTED by design (phases 24-26 pattern).
Output: `Profile.SpoofCanvas`; seeded `scriptCanvasNoise(seed)` / `scriptAudioContextNoise(seed)`; a real `WebGLManager` canvas-noise patch on toDataURL + getImageData; `EvasionManager.noiseSeed` + `SetNoiseSeed` + Apply() threading & gating.
</objective>
