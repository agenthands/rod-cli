<objective>
Add the two Phase-27 hardening toggles to the stealth config surface: `WebRTCLeakProtection` and `CanvasNoise` bool fields on `StealthConfig` (both **default true**), the `--webrtc-protection` and `--canvas-noise` CLI flags (urfave BoolFlag `Value:true`), forwarding into the daemon spawn, daemon-side capture into `StealthFlags`, and the `ResolveStealth` precedence so the no-flag path stays TRUE and an explicit `=false` flips it off.

Purpose: This is the config spine for both HARDEN-01 (WebRTC) and HARDEN-02 (canvas/audio noise). Plan 03 (runtime wiring) reads `cfg.Stealth.WebRTCLeakProtection` and `cfg.Stealth.CanvasNoise` to gate the actual evasion calls; this plan makes those fields exist, default-on, and overridable. Pure rod-cli config layer — no godoll, no browser.
Output: extended `StealthConfig` + `StealthFlags`, two new global BoolFlags + conditional forwarding in `cmd.go`, daemon-side capture in `main.go`, default-true + override logic in `ResolveStealth`.
</objective>
