<objective>
Build the offline, deterministic detection fixture: a `127.0.0.1:0` HTTP server (`internal/detect`) that serves a self-authored, `//go:embed`-bundled sannysoft-style detection page. The page runs the table-stakes signal probes plus informational WebRTC/CDP probes and writes per-signal verdicts into a known JS global (`window.__detect`) that a later e2e test reads back via the `eval` command.

Purpose: This is HARNESS-01 — the deterministic, offline backbone the entire v1.6 milestone (Phases 25–29) asserts against. It adds NO new evasion features; it is the measuring instrument.
Output: New `internal/detect` package — `server.go` (lifecycle), `embed.go` (embed glue), `detect.html` + `detect.js` (the page), `server_test.go` (server unit test).
</objective>

This is a self-authored sannysoft-style page — do NOT vendor or copy bot.sannysoft (no clean license, per D-CONTEXT decision "Detection page is self-authored").

Create `internal/detect/detect.js` that, on load, computes the table-stakes detection signals and writes a per-signal verdict object onto a single known global `window.__detect` (a flat object keyed by signal name, each value a string/bool the e2e test reads back via `eval`). Cover exactly these table-stakes signals (per HARNESS-01 + CONTEXT.md line 19):
  - `webdriver`: value of `navigator.webdriver`
  - `pluginsLength`: `navigator.plugins.length`
  - `mimeTypesLength`: `navigator.mimeTypes.length`
  - `userAgent`: `navigator.userAgent` (the e2e test asserts it does NOT contain `HeadlessChrome`)
  - `webglVendor` / `webglRenderer`: read via a WebGL context + `WEBGL_debug_renderer_info` extension, querying `UNMASKED_VENDOR_WEBGL` (37445) and `UNMASKED_RENDERER_WEBGL` (37446); guard for null context/extension
  - `permissionsConsistent`: compare `Notification.permission` against `navigator.permissions.query({name:'notifications'})` state (the classic headless mismatch tell); resolve the async query into the global before signalling ready
  - `languages`: `navigator.languages` joined
  - `screen`: `screen.width` + "x" + `screen.height` (plus `window.outerWidth`/`outerHeight` so the headless zero-size tell is visible)
  - `windowChrome`: boolean `typeof window.chrome !== 'undefined'` and `typeof (window.chrome && window.chrome.runtime)`
  - `timezone`: `Intl.DateTimeFormat().resolvedOptions().timeZone`
Then add INFORMATIONAL, non-blocking probes (CONTEXT.md line 25 — these are recorded but NOT asserted as blocking; they feed the CDP findings note in plan 24-04):
  - `webrtcIce`: attempt an `RTCPeerConnection` ICE-candidate gather; record any host/local IP string found (or empty). This is the KNOWN-RED WebRTC leak signal — record current truth, do not fix it (HARDEN-01 is Phase 27).
  - `cdpTell`: a lightweight CDP-presence heuristic (e.g. record whether the `Runtime.enable` error-serialization / stack-getter trick fires) into the global as an informational string.
Because the permissions and WebRTC probes are async, set a final `window.__detect.ready = true` only after they settle (use a short timeout fallback so the global is always populated), so the e2e test can poll for readiness. Keep the JS dependency-free (no external fetch, no CDN) to preserve the offline + single-binary constraint.

Wrap probe bodies in try/catch and record the error string into the corresponding signal rather than throwing, so one failing probe never blanks the whole global.

Create `internal/detect/server.go` as `package detect`, mirroring `internal/plugin/scanner/testserver/server.go` exactly: a `DetectServer` struct holding `listener net.Listener` and `mux *http.ServeMux`; `New() (*DetectServer, error)` that does `net.Listen("tcp", "127.0.0.1:0")` (loopback only — never bind a public interface, per the phase threat model), builds the mux, calls `registerRoutes()`, and returns; `Start()` that runs `go http.Serve(s.listener, s.mux)`; `Close()` that closes the listener; `URL() string` that returns `fmt.Sprintf("http://%s", s.listener.Addr().String())`.

`registerRoutes()` registers:
  - `GET /` → handler that sets `Content-Type: text/html; charset=utf-8` and writes the embedded `detectHTML`.
  - `GET /detect.js` → handler that sets `Content-Type: application/javascript; charset=utf-8` and writes the embedded `detectJS`.
  - `POST /result` (OPTIONAL scorecard sink, per PATTERNS.md line 48 / STACK.md line 53) → decode a JSON body into a small `Scorecard` struct and store it under a mutex; expose it via a `LastScorecard()` accessor. Only add this if it stays trivial; the e2e test in plan 24-03 reads `window.__detect` via `eval` and does NOT require the POST path, so keep `/result` minimal or omit it cleanly.

Do not serve any untrusted user input — all responses are static embedded assets (threat model: no reflected input, loopback bind, offline).

Create `internal/detect/server_test.go` (`package detect`) as a stdlib unit test: `New()` the server, `Start()`, `defer Close()`, `http.Get(s.URL())` the root, assert 200 + body contains a known marker from detect.html (e.g. the `<title>`), then `http.Get(s.URL()+"/detect.js")` and assert the body contains `window.__detect`. This proves the offline server + embed wiring without a browser.
