## Objective
Expose CLI commands and daemon handlers to manage plugin lifecycle.

## Requirements Covered
- PLUG-07
- PLUG-08
- PLUG-09

## Tasks

<task>
<action>
Update `types/context.go` to include `PluginEngine *plugin.PluginEngine` and initialized plugins list.
Create `actions/plugin.go` with functions `PluginLoad(ctx *types.Context, path string)`, `PluginList(ctx *types.Context)`, and `PluginRun(ctx *types.Context, name string)`.
In `daemon/daemon.go`'s `executeAction` switch statement, add handling for `plugin-load`, `plugin-list`, and `plugin-run` which route to the new functions in `actions/plugin.go`.
In `cmd.go`, add a new `plugin` command with `load`, `list`, and `run` subcommands that call `runClientCommand` with the respective daemon commands.
</action>
<acceptance_criteria>

- `rod-cli plugin run <name>` manually triggers a named plugin script.
</acceptance_criteria>
</task>
