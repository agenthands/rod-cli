## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| profile.UserAgent -> Sec-Ch-Ua header | Profile UA string (ultimately user-controllable via rod-cli flags) is parsed and reflected into an HTTP request header |
| profile.UserAgent -> injected JS (userAgentData) | Derived major version is interpolated into a JS string injected into the page |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-04 | Tampering | `chromeMajorFromUA` -> Sec-Ch-Ua header | mitigate | The regexp extracts ONLY `\d+` (digits) from the UA; the major is purely numeric, so no attacker-controlled substring (quotes, CRLF) can reach the header value — header injection is structurally impossible. Non-matching UA falls back to the numeric default `121`. |
| T-26-05 | Tampering | derived major -> scriptMockUserAgentData JS string | mitigate | The interpolated `version` is the same digits-only value; it cannot break out of the JS string literal or inject script. (platform was already interpolated; this adds no new untrusted-string sink — both are constrained values.) |
| T-26-06 | Information Disclosure | derived Timezone | accept | Timezone is non-sensitive and locale-derived from a fixed lookup; no PII or network/geo-IP lookup is introduced (offline-deterministic preserved). |
| T-26-SC | Tampering | go module deps | mitigate | No new third-party packages; only stdlib `regexp`/`fmt`/`strconv` and existing godoll code. Any new dep requires a blocking human-verify checkpoint + audit first. |
</threat_model>
