## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stealth-check url arg -> page.Navigate | User-supplied URL is navigated to before probing |
| live page -> stdout | Per-signal values (incl. UA) read from the page are formatted into stdout |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-10 | Tampering | injected probe JS | mitigate | The probe is a fixed embedded `//go:embed` string with no interpolation of user input — it cannot be tampered via the url arg or signal values. The action only READS window.__detect (string-coerced), it does not eval attacker-controlled script. |
| T-26-11 | Information Disclosure | --raw / --json stdout for piped callers | mitigate | --raw emits only PASS/FAIL + failing-signal names; no full page content or env is dumped. Warnings/errors go to log/stderr, keeping stdout clean (token-efficiency + no incidental leakage). |
| T-26-12 | Spoofing | reading verdict from a Go field instead of the live page | mitigate | StealthCheck reads every signal via window.__detect from the LIVE page (carried-forward validate-live-not-source rule); Plan 05 harness asserts this end-to-end. |
| T-26-SC | Tampering | terser/npm for JS minify | mitigate | If terser is used to minify probe_raw.js->probe.js, it is the existing dev-dependency already in package.json (no new install). Embedding the hand-written probe.js directly avoids any install. Any NEW package requires a blocking checkpoint + audit. |
</threat_model>
