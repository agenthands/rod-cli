<objective>
Build the configuration + validation layer for the pinnable, coherent fingerprint: extend StealthConfig with the reserved identity fields, register the 4 curated override flags (`--user-agent`, `--locale`, `--timezone`, `--platform`) and the `stealth-check` subcommand, forward the non-secret flags into the daemon spawn, capture them daemon-side, and implement the consistency validator inside `ResolveStealth()` (derive-when-unset, reject-when-user-conflicts) so a contradiction fails fast on stderr before the browser launches.

Purpose: This is the resolution/validation spine of the phase. The UA string is the single derivation anchor; this plan turns flags + profile into a validated, coherent `cfg.Stealth` that downstream plans (03 runtime wiring, 05 harness) consume. (FINGERPRINT-01, FINGERPRINT-02; registers the surface for VALIDATE-01/02.)
Output: Extended `StealthConfig`/`StealthFlags`, a `deriveAndValidateFingerprint` validator + a UA->major-version helper in `types/config.go`, 4 new global flags + forwarding + `stealth-check` registration in `cmd.go`, daemon-side flag capture in `main.go`.
</objective>
