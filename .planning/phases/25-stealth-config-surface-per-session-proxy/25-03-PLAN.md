| concurrent sessions sa/sb | Two daemons, two browsers, two proxies. The isolation test is the gate proving no shared-daemon bleed (the resolved Phase 25/26 architectural concern). |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-09 | Information Disclosure | proxy creds in .port file / default output | mitigate | `TestProxyCredentialsNotLeaked` reads the `.port` file + folded stdout/stderr and asserts no credential substring — this is the executable proof of T-25-01/T-25-02 from Plans 01/02. |
| T-25-10 | Tampering/Spoofing | session config bleed between concurrent sessions | mitigate | `TestProxySessionIsolation` asserts each session's egress goes through its OWN proxy and zero requests hit the other fixture — the executable proof per-session isolation holds. |
