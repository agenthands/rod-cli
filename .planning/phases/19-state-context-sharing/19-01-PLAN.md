# Plan 19-01: State Context API

## Objective
Provide a secure API for plugins to read the DOM snapshot and network state from the active `rod.Page`.

## Requirements Covered
- PLUG-10
- PLUG-11

## Tasks

<task>
<read_first>
- internal/plugin/engine.go
</read_first>
<action>
Create `internal/plugin/api.go`.
Define `PluginAPI` struct containing a reference to the active `*rod.Page`.
Implement `GetCookies() (interface{}, error)` on `PluginAPI` that calls `page.Browser().GetCookies()`.
Implement `GetSnapshot() (string, error)` on `PluginAPI` that calls `page.HTML()`.
In `internal/plugin/engine.go`, update `BindLifecycle(ctx context.Context, page *rod.Page)` to inject an instance of `PluginAPI` into the `goja.Runtime` under the global variable `api`, so scripts can call `api.GetCookies()`.
</action>
<acceptance_criteria>
- `internal/plugin/api.go` exists.
- `PluginAPI` exposes `GetCookies` and `GetSnapshot`.
- `PluginAPI` is injected into the `goja.Runtime` as `api`.
</acceptance_criteria>
</task>

## Verification
<must_haves>
<truths>
- Plugin scripts can query the token-optimized snapshot tree.
- Plugin scripts can read active cookies and local storage via standard API mappings.
</truths>
</must_haves>
