<objective>
Thread godoll's existing `humanize.With*` options through the rod-cli config spine
and into the `actions/actions.go` call sites, tunable via CLI flags and the named
JSON profile (HUMANIZE-01). This is the full implementation slice: config fields,
flags + forwarding + capture, `ResolveStealth` precedence, a `Context` accessor,
and the threaded variadic calls. Plan 02 proves tunability + zero-regression.

Purpose: `actions.go` currently calls `TypeWithHumanize`/`ClickWithMouse`/`ScrollBy`
with defaults only. After this plan the resolved `cfg.Stealth` humanize knobs reach
those call sites; unset knobs emit no options (godoll default → zero regression).
Output: extended `StealthConfig`+`StealthFlags`, humanize flags in `cmd.go`+`main.go`,
`ResolveStealth` precedence, a `*Context` accessor, and threaded calls in `actions.go`.
</objective>
